## À propos de l'assistant

L'assistant TockDocs est une interface de chat **ancrée dans votre documentation** construite sur :

- **AI SDK** pour l'orchestration du chat et du modèle
- **MCP** pour les outils documentaires
- **FlexSearch** pour la recherche plein texte principale
- **Fuse.js** pour la recherche fuzzy de secours

Il est conçu pour répondre à partir de votre documentation, et non uniquement depuis la mémoire générique du modèle.

Quand un utilisateur pose une question, l'assistant :

- **recherche et récupère** le contenu pertinent via le [serveur MCP](/docs/platform/fr/ai/mcp)
- **cherche dans le contenu complet des pages**, pas seulement dans les titres
- **utilise un fallback fuzzy** quand la recherche exacte est faible
- **cite ses sources** avec des liens vers les pages
- **peut générer des exemples de code** ancrés dans la documentation récupérée

## Comment ça fonctionne

Le flux runtime est orienté outils :

1. Le client envoie les messages à `/__tockdocs__/assistant`.
2. Le serveur résout le provider et le modèle actifs.
3. Quand la requête provient d'une page de docs, TockDocs tente de la restreindre à la **knowledge base** et à la **locale** en cours.
4. L'assistant expose des outils MCP au modèle.
5. `search-pages` effectue la récupération avec FlexSearch puis Fuse.js en fallback si nécessaire.
6. Le modèle renvoie une réponse ancrée en streaming vers l'UI.

Par défaut, l'assistant se connecte au serveur MCP intégré à `/mcp`, qui expose `search-pages`, `list-pages` et `get-page`.

## Démarrage rapide

### 1. Configurer un provider

TockDocs prend en charge **Vercel AI Gateway**, ainsi que :

- OpenRouter
- DeepSeek
- Nvidia
- Hugging Face
- Groq
- GitHub Models
- Gemini
- Cloudflare Workers AI

**Vercel AI Gateway**

```bash [.env]
AI_PROVIDER=vercel
AI_MODEL=google/gemini-3-flash
# Utilisez AI_GATEWAY_API_KEY pour les clés manuelles ; Vercel réserve le préfixe VERCEL_*.
AI_GATEWAY_API_KEY=votre-cle-api
```

**OIDC (uniquement sur Vercel)** — `VERCEL_OIDC_TOKEN` est injecté automatiquement. Rien à ajouter en production. En local, exécutez `vercel env pull` sur un [projet lié](https://vercel.com/docs/cli/link).

**Exemple avec un autre provider**

```bash [.env]
AI_PROVIDER=deepseek
DEEPSEEK_API_KEY=votre-cle-api
DEEPSEEK_MODEL=deepseek-chat
```

Si `AI_PROVIDER` n'est pas défini, TockDocs détecte automatiquement le premier provider configuré à partir des credentials disponibles.

### 2. Lancez ou déployez votre site

En développement, l'UI de l'assistant est activée automatiquement.

En production, elle devient disponible quand :

- `NUXT_PUBLIC_ASSISTANT_ENABLED=true`, ou
- des credentials de provider pris en charge sont présents

## Utiliser l'assistant

### Input flottant

Sur les pages de documentation, un input flottant apparaît en bas de l'écran.

::tip
Utilisez le raccourci :kbd{value="meta"} :kbd{value="I"} pour focaliser l'input flottant.
::

### Explain with AI

Chaque page de documentation inclut un bouton **Explain with AI** dans le sommaire latéral. En cliquant dessus, l'assistant s'ouvre avec la page courante comme contexte.

### Slideover de chat

Quand une conversation commence, un panneau latéral s'ouvre à droite et conserve l'historique visible.

## Configuration UI

Configurez l'UI de l'assistant dans `app.config.ts` :

```ts [app/app.config.ts]
export default defineAppConfig({
  assistant: {
    floatingInput: true,
    explainWithAi: true,
    faqQuestions: [],
    shortcuts: {
      focusInput: 'meta_i',
    },
    icons: {
      trigger: 'i-lucide-sparkles',
      explain: 'i-lucide-brain',
    },
  },
})
```

### FAQ Questions

Affichez des questions suggérées quand le chat est vide.

#### Format simple

```ts [app/app.config.ts]
export default defineAppConfig({
  assistant: {
    faqQuestions: [
      'Comment installer TockDocs ?',
      'Comment personnaliser le thème ?',
      'Comment ajouter des composants à mes pages ?',
    ],
  },
})
```

#### Format catégorisé

```ts [app/app.config.ts]
export default defineAppConfig({
  assistant: {
    faqQuestions: [
      {
        category: 'Démarrage',
        items: ['Comment installer TockDocs ?', 'Quelle est la structure du projet ?'],
      },
      {
        category: 'Personnalisation',
        items: ['Comment changer les couleurs ?', 'Comment ajouter un logo ?'],
      },
    ],
  },
})
```

#### Format localisé

```ts [app/app.config.ts]
export default defineAppConfig({
  assistant: {
    faqQuestions: {
      en: [{ category: 'Getting Started', items: ['How do I install?'] }],
      fr: [{ category: 'Démarrage', items: ['Comment installer ?'] }],
    },
  },
})
```

export default defineAppConfig({

## Configuration avancée

Configurez les options runtime avancées dans `nuxt.config.ts` sous `tockdocs.assistant` :

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  tockdocs: {
    assistant: {
      provider: 'vercel',
      model: 'google/gemini-3-flash',
      mcpServer: '/mcp',
      apiPath: '/__tockdocs__/assistant',
    },
  },
})
```

::warning
L'ancienne configuration top-level `assistant` est encore lue pour compatibilité, mais elle est dépréciée. Préférez `tockdocs.assistant`.
::

### Choix du serveur MCP

Vous pouvez :

- utiliser le serveur MCP intégré à `/mcp`
- pointer vers un serveur MCP externe

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  tockdocs: {
    assistant: {
      mcpServer: 'https://autre-docs.exemple.com/mcp',
    },
  },
})
```

### Surcharge du provider et du modèle

Vous pouvez fixer explicitement un provider :

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  tockdocs: {
    assistant: {
      provider: 'deepseek',
    },
  },
})
```

Et surcharger le modèle utilisé par ce provider :

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  tockdocs: {
    assistant: {
      model: 'anthropic/claude-opus-4.5',
    },
  },
})
```

## Désactiver des fonctionnalités

### Désactiver l'input flottant

```ts [app/app.config.ts]
export default defineAppConfig({
  assistant: {
    floatingInput: false,
  },
})
```

### Désactiver « Explain with AI »

```ts [app/app.config.ts]
export default defineAppConfig({
  assistant: {
    explainWithAi: false,
  },
})
```

### Désactiver entièrement l'assistant

L'UI de l'assistant est désactivée lorsqu'aucun credential de provider pris en charge n'est disponible et que `NUXT_PUBLIC_ASSISTANT_ENABLED` n'est pas défini à `true`.

```bash [.env]
NUXT_PUBLIC_ASSISTANT_ENABLED=false
```

## Accès programmatique

Utilisez le composable `useAssistant` pour piloter l'assistant par code :

```vue
<script setup>
const { isEnabled, isOpen, open, close, toggle } = useAssistant()

function askQuestion() {
  open('Comment configurer le thème ?', true)
}
</script>

<template>
  <UButton v-if="isEnabled" @click="askQuestion">
    Poser une question sur le thème
  </UButton>
</template>
```

### API du composable

| Propriété                        | Type                   | Description |
| -------------------------------- | ---------------------- | ----------- |
| `isEnabled`                      | `ComputedRef<boolean>` | Indique si l'UI de l'assistant est activée dans le runtime courant |
| `isOpen`                         | `Ref<boolean>`         | Indique si le slideover est ouvert |
| `open(message?, clearPrevious?)` | `Function`             | Ouvre l'assistant, éventuellement avec un message |
| `close()`                        | `Function`             | Ferme le slideover |
| `toggle()`                       | `Function`             | Ouvre/ferme l'assistant |
| `clearMessages()`                | `Function`             | Efface l'historique de conversation |
