## À propos des serveurs MCP

Le [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) est un protocole ouvert qui permet de connecter des applications IA à des outils et des sources de données externes.

Chaque instance TockDocs inclut un serveur MCP intégré afin que des outils comme Claude, Cursor, VS Code, Windsurf ou Zed puissent rechercher et lire votre documentation.

## Accéder à votre serveur MCP

Votre serveur MCP est disponible au chemin `/mcp` de l'URL de votre documentation.

::note
Si votre documentation est hébergée sur `https://docs.example.com`, l'URL MCP est `https://docs.example.com/mcp`.
::

## Désactiver le serveur MCP

Si vous souhaitez désactiver le serveur MCP, faites-le dans `nuxt.config.ts` :

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  mcp: {
    enabled: false,
  },
})
```

## Outils intégrés

TockDocs expose trois outils intégrés.

### `search-pages`

Recherche dans les titres, descriptions, headings, chemins et le contenu complet des pages.

| Paramètre | Type              | Description |
| --------- | ----------------- | ----------- |
| `query`   | string (requis)   | Requête de recherche ou question en langage naturel |
| `limit`   | number (optionnel) | Nombre maximum de résultats |
| `kb`      | string (optionnel) | Id de knowledge base pour restreindre la recherche sur les sites multi-KB |
| `locale`  | string (optionnel) | Code locale pour restreindre la recherche |

`search-pages` utilise FlexSearch en premier passage puis Fuse.js en fallback fuzzy.

### `list-pages`

Liste les pages disponibles avec leur titre, chemin, description et URL.

| Paramètre | Type               | Description |
| --------- | ------------------ | ----------- |
| `kb`      | string (optionnel) | Id de knowledge base pour restreindre les résultats sur les sites multi-KB |
| `locale`  | string (optionnel) | Code locale pour restreindre les résultats |

### `get-page`

Récupère le contenu Markdown complet d'une page de documentation précise.

| Paramètre | Type             | Description |
| --------- | ---------------- | ----------- |
| `path`    | string (requis)  | Chemin exact de la page |

Exemples :

- mode KB : `/docs/platform/en/getting-started/installation`
- mode legacy i18n : `/en/getting-started/installation`
- mode legacy monolingue : `/getting-started/installation`

## Comportement orienté KB

Sur les sites multi-KB :

- les clients MCP externes peuvent passer `kb` et `locale` explicitement à `search-pages` et `list-pages`
- l'assistant intégré peut aussi déduire la KB et la locale courantes depuis la page de docs qui a déclenché la requête

Cela permet de restreindre la récupération au bon ensemble documentaire lorsque c'est possible.

## Installation

Le serveur MCP TockDocs utilise un transport HTTP et peut être connecté à plusieurs assistants.

### Claude Code

```bash
claude mcp add --transport http my-docs https://docs.example.com/mcp
```

### Cursor

:install-button{ide="cursor" label="Installer dans Cursor" url="https://docs.example.com/mcp"}

Ou créez/modifiez `.cursor/mcp.json` :

```json [.cursor/mcp.json]
{
  "mcpServers": {
    "my-docs": {
      "type": "http",
      "url": "https://docs.example.com/mcp"
    }
  }
}
```

### Visual Studio Code

:install-button{ide="vscode" label="Installer dans VS Code" url="https://docs.example.com/mcp"}

Ou créez/modifiez `.vscode/mcp.json` :

```json [.vscode/mcp.json]
{
  "servers": {
    "my-docs": {
      "type": "http",
      "url": "https://docs.example.com/mcp"
    }
  }
}
```

### Windsurf

```json [.codeium/windsurf/mcp_config.json]
{
  "mcpServers": {
    "my-docs": {
      "type": "http",
      "url": "https://docs.example.com/mcp"
    }
  }
}
```

### Zed

```json [.config/zed/settings.json]
{
  "context_servers": {
    "my-docs": {
      "source": "custom",
      "command": "npx",
      "args": ["mcp-remote", "https://docs.example.com/mcp"],
      "env": {}
    }
  }
}
```

## Personnalisation

TockDocs utilise `@nuxtjs/mcp-toolkit`, vous pouvez donc étendre le serveur MCP avec des outils, ressources, prompts et handlers personnalisés.

### Ajouter des outils

```ts [server/mcp/tools/search.ts]
import { z } from 'zod'

export default defineMcpTool({
  description: 'Search documentation by keyword',
  inputSchema: {
    query: z.string().describe('The search query'),
  },
  handler: async ({ query }) => {
    const results = await searchDocs(query)
    return {
      content: [{ type: 'text', text: JSON.stringify(results) }],
    }
  },
})
```

### Ajouter des ressources

```ts [server/mcp/resources/changelog.ts]
export default defineMcpResource({
  file: 'CHANGELOG.md',
  metadata: {
    description: 'Project changelog',
  },
})
```

### Ajouter des prompts

```ts [server/mcp/prompts/migration-help.ts]
import { z } from 'zod'

export default defineMcpPrompt({
  description: 'Get help with migrating between versions',
  inputSchema: {
    fromVersion: z.string().describe('Current version'),
    toVersion: z.string().describe('Target version'),
  },
  handler: async ({ fromVersion, toVersion }) => {
    return {
      messages: [
        {
          role: 'user',
          content: {
            type: 'text',
            text: `Help me migrate from version ${fromVersion} to ${toVersion}. What changed?`,
          },
        },
      ],
    }
  },
})
```

### Ajouter des handlers séparés

Vous pouvez exposer différents endpoints MCP avec leurs propres ensembles d'outils.

```ts [server/mcp/migration.ts]
import { z } from 'zod'

const migrationTool = defineMcpTool({
  name: 'migrate-v3-to-v4',
  description: 'Migrate code from version 3 to version 4',
  inputSchema: {
    code: z.string().describe('The code to migrate'),
  },
  handler: async ({ code }) => {
    return {
      content: [{ type: 'text', text: code }],
    }
  },
})

export default defineMcpHandler({
  route: '/mcp/migration',
  name: 'Migration Assistant',
  version: '1.0.0',
  tools: [migrationTool],
})
```
