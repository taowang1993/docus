## TockDocs est une app Nuxt

TockDocs est distribué comme un **layer Nuxt**. Quand vous générez un projet, vous obtenez une application Nuxt normale qui étend le layer et peut utiliser tout l'écosystème Nuxt standard.

Un starter ressemble généralement à ceci :

```bash
my-docs/
├── content/
├── nuxt.config.ts
├── package.json
└── public/
```

Ensuite, vous pouvez ajouter les répertoires Nuxt habituels comme `app/`, `server/`, `plugins/` ou `modules/`.

## Modules Nuxt

Vous pouvez installer et configurer des modules Nuxt exactement comme dans n'importe quel autre projet Nuxt.

Exemple : ajouter Vercel Analytics.

::steps

### Installez le module

```bash [Terminal]
npm install @vercel/analytics
```

### Activez-le dans `nuxt.config.ts`

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['tockdocs'],
  modules: ['@vercel/analytics/nuxt/module'],
})
```
::

## Composants Vue personnalisés dans les docs

Vous pouvez utiliser Markdown + MDC pour la plupart des interfaces docs, mais vous pouvez aussi créer vos propres composants Vue et les utiliser dans le contenu.

Exemple de composant :

::tabs
:::tabs-item{.my-5 icon="i-lucide-code" label="Composant Vue"}
```vue [app/components/content/BrowserFrame.vue]
<script setup lang="ts">
defineProps<{
  title?: string
}>()
</script>

<template>
  <div class="w-fit overflow-hidden rounded-xl border border-muted bg-accented px-2 pb-2 shadow-md">
    <div class="flex items-center justify-between border-b border-accented bg-accented px-2 py-2">
      <div class="flex items-center gap-2">
        <span class="h-3 w-3 rounded-full bg-red-500" />
        <span class="h-3 w-3 rounded-full bg-yellow-500" />
        <span class="h-3 w-3 rounded-full bg-green-500" />
      </div>
      <div class="text-muted">
        {{ title }}
      </div>
    </div>
    <slot mdc-unwrap="p" />
  </div>
</template>
```

## Pages Vue

En plus des pages Markdown, vous pouvez créer des pages Vue classiques.

```vue [app/pages/hello.vue]
<template>
  <div>
    <h1>Hello</h1>
  </div>
</template>
```

Vous pouvez aussi utiliser `definePageMeta` pour changer le layout ou masquer les éléments du shell par défaut :

```vue [app/pages/hello.vue]
<script setup lang="ts">
definePageMeta({
  layout: 'default',
  header: false,
  footer: false,
})
</script>
```

## Layouts personnalisés

TockDocs fournit :

- `default` pour les landing pages et les pages Vue personnalisées
- `docs` pour les pages de documentation

Vous pouvez ajouter vos propres layouts dans `app/layouts/` :

```vue [app/layouts/custom.vue]
<template>
  <main class="custom-layout">
    <slot />
  </main>
</template>
```

## Pourquoi le layer est important

Comme TockDocs est un layer et non un template fermé, vous pouvez :

- garder la rédaction des docs en Markdown
- surcharger les composants intégrés quand nécessaire
- ajouter vos propres pages et APIs
- installer d'autres modules Nuxt
- mélanger pages marketing, pages applicatives et docs dans un seul projet
