TockDocs repose sur **Nuxt Content**. Vous rédigez vos docs en Markdown, vous ajoutez une UI plus riche avec la **syntaxe MDC**, et le layer dérive le routage, la navigation, la recherche et les métadonnées de page à partir de votre structure de contenu.

## Modes de contenu

TockDocs prend en charge deux architectures de contenu et de routage.

### Mode legacy

Le mode legacy est la structure par défaut des starters générés.

Exemple monolingue :

```bash
content/
├── index.md
└── 1.getting-started/
    └── 3.installation.md
```

Exemple localisé :

```bash
content/
├── en/
│   ├── index.md
│   └── 1.getting-started/
└── fr/
    ├── index.md
    └── 1.getting-started/
```

Les routes sont dérivées depuis l'arborescence :

- `content/index.md` → `/`
- `content/1.getting-started/3.installation.md` → `/getting-started/installation`
- `content/en/1.getting-started/3.installation.md` → `/en/getting-started/installation`

Le mode legacy prend aussi en charge un sous-dossier optionnel `content/docs/` pour les apps qui veulent exposer la documentation sous `/docs`.

### Mode knowledge base

Le mode KB est activé lorsque TockDocs détecte un ou plusieurs fichiers `content/<kb>/kb.yml`.

```bash
content/
├── site/
│   └── index.md
└── platform/
    ├── kb.yml
    └── en/
        └── 1.getting-started/
            └── 3.installation.md
```

Les routes deviennent orientées KB :

- `content/site/index.md` → `/`
- `content/platform/en/1.getting-started/3.installation.md` → `/docs/platform/en/getting-started/installation`

Le fichier `kb.yml` définit l'id de la KB, son titre, ses locales, sa locale par défaut, sa page d'entrée et d'autres métadonnées UI.

## Comportement de la page d'accueil

### Mode legacy

S'il n'existe pas de landing page Vue personnalisée, TockDocs utilise le contenu Markdown :

- `content/index.md` pour les apps monolingues
- `content/<locale>/index.md` pour les apps i18n

### Mode knowledge base

En mode KB, la racine `/` se comporte différemment :

- si `content/site/index.md` existe, cette page est affichée à `/`
- sinon TockDocs affiche un répertoire des knowledge bases

### Landing page Vue personnalisée

Comme TockDocs est un layer Nuxt, vous pouvez toujours remplacer la page d'accueil par une page Vue :

```vue [app/pages/index.vue]
<template>
  <main>
    <h1>Landing page personnalisée</h1>
  </main>
</template>
```

Quand `app/pages/index.vue` existe, Nuxt gère `/` avec votre page au lieu du template de landing par défaut.

## MDC dans le Markdown

MDC vous permet d'utiliser des composants Vue directement dans le Markdown.

### Composants

```mdc
::note
Bonjour depuis MDC
::
```

### Slots

```mdc
:::u-page-feature
  #title
  Nuxt 4

  #description
  Powered by Nuxt et Nuxt Content.
:::
```

### Props

Props inline :

```mdc
::card{title="Docs" icon="i-lucide-book-open"}
Une carte rendue depuis Markdown.
::
```

Props via frontmatter YAML dans le bloc du composant :

```mdc
::card
---
title: Docs
icon: i-lucide-book-open
---
Une carte rendue depuis Markdown.
::
```

::tip{to="https://content.nuxt.com/docs/files/markdown"}
Consultez la documentation Nuxt Content pour la référence complète de la syntaxe MDC.
::

## Frontmatter

Chaque page de contenu peut définir des métadonnées en haut du fichier :

```md [content/getting-started/edition.md]
---
title: Édition
description: Apprenez à rédiger votre documentation.
---

Contenu de la page.
```

Clés courantes utilisées par les pages TockDocs :

| Clé           | Type                                     | Description |
| ------------- | ---------------------------------------- | ----------- |
| `title`       | `string`                                 | Titre de la page affiché dans l'en-tête et utilisé comme fallback SEO. |
| `description` | `string`                                 | Description affichée dans l'en-tête et utilisée comme fallback SEO. |
| `navigation`  | `boolean \| object`                     | Contrôle l'apparition de la page dans la navigation. |
| `layout`      | `string`                                 | Permet de surcharger le layout Nuxt. |
| `seo`         | `{ title?: string, description?: string, ogImage?: string }` | Surcharges SEO par page. |
| `sitemap`     | `boolean`                                | Exclut la page du sitemap quand la valeur est `false`. |

## Fichiers ordonnés et slugs publics

TockDocs ignore les préfixes numériques lors de la génération des URLs publiques. Par exemple :

```bash
content/
└── 1.getting-started/
    └── 3.installation.md
```

produit :

- `/getting-started/installation` en mode legacy
- `/docs/platform/en/getting-started/installation` en mode KB

Utilisez les préfixes numériques pour contrôler l'ordre dans la navigation sans modifier le slug public.

## Ce que TockDocs dérive automatiquement

À partir de votre structure de contenu, TockDocs dérive automatiquement :

- les URLs des pages
- la navigation de gauche
- le sommaire de la page
- l'index de recherche
- les données de grounding de l'assistant
- les entrées du sitemap
- les collections par locale et par KB en mode knowledge base
