## CLI `create-tockdocs`

::steps

### Créez votre répertoire de docs

Utilisez le CLI `create-tockdocs` pour générer un nouveau projet TockDocs :

```bash [Terminal]
npx create-tockdocs my-docs
```

Vous pouvez choisir entre deux templates :

- `default` — documentation monolingue en mode legacy
- `i18n` — documentation multilingue en mode legacy avec `@nuxtjs/i18n`

```bash [Terminal]
# Créer avec le starter i18n
npx create-tockdocs my-docs -t i18n
```

### Lancez le serveur de développement

Placez-vous dans le dossier du projet et démarrez Nuxt en développement :

```bash [Terminal]
cd my-docs
npm run dev
```

Par défaut, TockDocs tourne sur <http://localhost:4987>.

::note
Si le port `4987` est déjà utilisé, le lanceur de développement s'arrête au lieu de basculer vers `3000` ou `3001`.
::

### Commencez à écrire

Consultez le guide [Édition](/docs/platform/fr/concepts/edition) pour comprendre le contenu, le routage, le frontmatter et la syntaxe MDC dans TockDocs.
::

## Quelle architecture utilisent les starters ?

Les starters générés restent volontairement simples :

- le starter `default` utilise le **mode legacy** avec un seul arbre de contenu
- le starter `i18n` utilise le **mode legacy + i18n** avec des dossiers comme `content/en` et `content/fr`

TockDocs prend aussi en charge le **mode knowledge base** pour les sites multi-KB comme la documentation officielle. Dans ce cas, le contenu est organisé sous `content/<kb>/<locale>/...` avec un fichier `kb.yml` par knowledge base.

::tip{to="/docs/platform/fr/getting-started/project-structure"}
Consultez le guide de structure du projet pour voir les exemples côte à côte entre mode legacy et mode KB.
::

## Skill pour assistant IA

Vous pouvez ajouter la documentation TockDocs comme skill à votre assistant IA (Cursor, Claude, etc.) :

```bash [Terminal]
npx skills add https://tockdocs.dev
```

Ce skill aide votre assistant à :

- générer des pages avec une structure TockDocs valide
- utiliser correctement MDC et les composants prose
- suivre les conventions de contenu et d'architecture
- appliquer les patterns de configuration actuels de TockDocs

::tip{to="/docs/platform/fr/ai/skills"}
Vous pouvez aussi publier vos propres skills depuis votre site TockDocs.
::

## Intégration du layer

TockDocs est un layer Nuxt. Le starter génère déjà un `nuxt.config.ts` qui étend `tockdocs` :

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['tockdocs'],
})
```

En pratique, vous utilisez ensuite les scripts Nuxt classiques de votre app :

```bash [Terminal]
npm run dev
npm run build
```
