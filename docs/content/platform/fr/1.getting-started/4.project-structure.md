## Vue d'ensemble

TockDocs est un **layer Nuxt**. Votre projet est donc une application Nuxt classique qui étend le layer et ajoute son propre contenu, sa configuration et, si besoin, son code Vue personnalisé.

Aujourd'hui, TockDocs prend en charge deux architectures de contenu :

- **mode legacy** — un seul arbre de documentation, éventuellement localisé avec `@nuxtjs/i18n`
- **mode knowledge base** — plusieurs ensembles de documentation sous `/docs/<kb>/<locale>/...`

Les starters générés utilisent le **mode legacy** par défaut. Le site officiel TockDocs utilise le **mode knowledge base**.

## Structures de base

### Starter `default`

Le starter `default` est une app monolingue en mode legacy :

```bash
my-docs/
├── content/
│   ├── index.md
│   └── 1.getting-started/
├── nuxt.config.ts
├── package.json
└── public/
```

Routes typiques :

- `content/index.md` → `/`
- `content/1.getting-started/3.installation.md` → `/getting-started/installation`

### Starter `i18n`

Le starter `i18n` reste en mode legacy, mais le contenu est organisé par locale :

```bash
my-docs/
├── content/
│   ├── en/
│   │   ├── index.md
│   │   └── 1.getting-started/
│   └── fr/
│       ├── index.md
│       └── 1.getting-started/
├── nuxt.config.ts
├── package.json
└── public/
```

Routes typiques :

- `content/en/index.md` → `/en`
- `content/fr/1.getting-started/3.installation.md` → `/fr/getting-started/installation`

### Mode knowledge base

Le mode KB est destiné aux sites multi-produits ou multi-domaines documentaires. Le site officiel utilise cette structure :

```bash
content/
├── site/
│   └── index.md
├── platform/
│   ├── kb.yml
│   ├── en/
│   │   └── 1.getting-started/
│   └── fr/
│       └── 1.getting-started/
└── parser/
    ├── kb.yml
    └── en/
        └── parser/
```

Routes typiques :

- `content/site/index.md` → `/`
- `content/platform/en/1.getting-started/3.installation.md` → `/docs/platform/en/getting-started/installation`
- `content/parser/en/parser/best-document-parsing-apis-2026.md` → `/docs/parser/en/parser/best-document-parsing-apis-2026`

Chaque KB est définie par un fichier `kb.yml` qui contrôle l'id, le titre, les locales, la locale par défaut, la page d'entrée et d'autres métadonnées UI.

::tip{to="/docs/platform/fr/concepts/edition"}
Consultez le guide d'édition pour les règles complètes de routage en mode legacy et en mode KB.
::

## `package.json`

Un starter TockDocs reste volontairement minimal. Le `package.json` généré ressemble à ceci :

```json [package.json]
{
  "name": "my-docs",
  "scripts": {
    "dev": "nuxt dev",
    "build": "nuxt build"
  },
  "dependencies": {
    "@takumi-rs/core": "^1.0.15",
    "better-sqlite3": "^12.6.2",
    "nuxt": "^4.3.1",
    "tockdocs": "latest"
  }
}
```

Notes :

- `tockdocs` fournit le layer Nuxt
- `nuxt` exécute votre application
- `better-sqlite3` est utilisé par Nuxt Content
- `@takumi-rs/core` est utilisé pour le rendu OG Takumi en développement local

## `nuxt.config.ts`

Le starter inclut déjà un fichier qui étend le layer :

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['tockdocs'],
})
```

C'est ici que vous ajoutez d'autres modules Nuxt, la config du site, i18n, ou les options avancées de l'assistant.

## `app/app.config.ts`

`app.config.ts` est optionnel, mais c'est l'endroit principal pour personnaliser l'UI TockDocs.

```ts [app/app.config.ts]
export default defineAppConfig({
  seo: {
    title: 'My Docs',
    description: 'Ma documentation',
  },
  header: {
    title: 'My Docs',
  },
})
```

Utilisez-le pour la marque, la navigation, les réseaux sociaux, le TOC, les liens GitHub, le mode couleur et les options UI de l'assistant.

## Les répertoires Nuxt standards restent disponibles

Comme TockDocs est un layer Nuxt, vous pouvez utiliser la structure Nuxt habituelle à côté de votre contenu :

```bash
my-docs/
├── app/
│   ├── app.config.ts
│   ├── components/
│   ├── layouts/
│   ├── pages/
│   └── composables/
├── content/
├── public/
├── server/
├── nuxt.config.ts
└── package.json
```

Utilisez ces répertoires pour :

- ajouter des composants Vue personnalisés
- surcharger les composants fournis par le layer
- créer des pages hors du contenu documentaire
- ajouter des routes API ou des utilitaires serveur
- installer d'autres modules Nuxt

::tip{to="/docs/platform/fr/concepts/nuxt"}
Consultez le guide Nuxt pour des exemples de composants personnalisés, pages Vue et layouts sur TockDocs.
::
