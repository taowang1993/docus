## `create-tockdocs` CLI

::steps

### Create your docs directory

Use the `create-tockdocs` CLI to create a new TockDocs project:

```bash [Terminal]
npx create-tockdocs my-docs
```

You can choose between two templates:

- `default`: Basic TockDocs setup for single-language documentation
- `i18n`: Includes internationalization support for multi-language documentation

```bash [Terminal]
# Create with i18n template
npx create-tockdocs my-docs -t i18n
```

We recommend using the `npm` package manager.

### Start your docs server in development

Move to your docs directory and start your docs server in development mode:

```bash [Terminal]
cd my-docs
npm run dev
```

A local preview of your documentation will be available at <http://localhost:3000>

### Write your documentation

Head over the [Edition](/docs/platform/en/concepts/edition) section to learn how to write your documentation.
::

## AI Assistant Skill

Get started quickly with TockDocs by adding specialized knowledge to your AI assistant (Cursor, Claude, etc.):

```bash [Terminal]
npx skills add https://tockdocs.dev
```

This skill helps you create documentation faster by providing your AI assistant with:

- Best practices for writing documentation with TockDocs
- MDC component usage and ready-to-use templates
- Writing guidelines and content structure patterns
- Configuration and customization tips

Once installed, your AI assistant can help you scaffold new documentation projects, generate pages with proper structure, and write content following TockDocs best practices.

::tip{to="/docs/platform/en/ai/skills"}
You can also publish your own skills from your TockDocs site. Learn more about Agent Skills.
::

## Layer Integration

TockDocs uses a **Nuxt layer-based approach**, you can extend the TockDocs layer directly in your `nuxt.config.ts` with `extends: ['tockdocs']`:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['tockdocs'],
})
```
